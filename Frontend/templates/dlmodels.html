Deep Neural Network models for Intrusion Detection

We were able to replicate results with our learnings from the EDA, and achieved similar performance metrics as ML models with only 10 epochs, achieving efficient training time over a very large dataset. To provide a more profound basis for our experiment, we used the Callbacks class from Keras, employing ModelCheckpoint to validate our findings.

A 3 layer network performs the best amongst the ones evaluated

Since data is actually very linear, 2 or 3 layers of a network would be able to find the deciding factors. Anything greater, variance in the model increases (which causes the model to lose performance since the network becomes more selective in feature weights)

We were able to reduce model size and training resources while maintaining a competent performance metric. Also, usage of ModelCheckpoint to reduce the overall number of weights we store per epoch for a DNN model model building allows us to deploy the IDS on most (if not all) mobile and routing devices, which was not previously possible due to the bulky nature of the models.

The ModelCheckpoint callback class allows you to define where to checkpoint the model weights, how the file should named and under what circumstances to make a checkpoint of the model. The API allows you to specify which metric to monitor, such as loss or accuracy on the training or validation dataset. You can specify whether to look for an improvement in maximizing or minimizing the score. Finally, the filename that you use to store the weights can include variables like the epoch number or metric.
